What's the difference between pre-pruning and post-pruning?
Pre-pruning stops tree growth early using criteria like max_depth, min_samples_split. Post-pruning grows full tree then removes branches that don't improve validation performance.
How does cost complexity pruning work?
Cost Complexity = Training Error + Œ± √ó Tree Complexity. Higher Œ± values create simpler trees by penalizing complexity. Uses cross-validation to find optimal Œ±.
What are common pre-pruning parameters?
max_depth, min_samples_split, min_samples_leaf, max_features, min_impurity_decrease. These limit tree growth to prevent overfitting.
How do you use cross-validation to optimize tree parameters?
Use GridSearchCV or RandomizedSearchCV with different parameter combinations. Evaluate on validation folds to find optimal hyperparameters that balance bias and variance.
